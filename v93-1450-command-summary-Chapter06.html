  • Prefer sudo over direct su for auditing
  • su - provides clean login environment
  • Monitor su usage in /var/log/secure
  • Disable direct root login for security
sudo
Execute commands as another user with detailed logging and controlled access
[security@server ~]$ sudo systemctl status ssh

üõ°Ô∏è Sudo Best Practices
  • All sudo activity logged to /var/log/secure
  • Use principle of least privilege
  • Configure timeout for security (default 5 minutes)
  • Prefer sudo over direct root access

üë• User Account Management
‚ñº
useradd
Create new user accounts with specified attributes and default settings
[root@security-server ~]# useradd analyst1
Create user with default settings
[root@security-server ~]# useradd -c "Security Analyst" -G security-ops -s /bin/bash analyst2
Create with comment, group, and shell
[root@security-server ~]# useradd -r -s /sbin/nologin monitoring
Create system account for service
UID 0
Root superuser account
UID 1-200
Static system accounts
UID 201-999
Dynamic system accounts
UID 1000+
Regular user accounts
üîê Account Creation Security
  • Use -r flag for system/service accounts
  • Set appropriate shell (/sbin/nologin for services)
  • Add users to security groups during creation
  • Document account purpose in comment field
usermod
Modify existing user accounts including groups, shell, home directory
[root@security-server ~]# usermod -aG wheel analyst1
Add user to wheel group (append mode)
[root@security-server ~]# usermod -L analyst2
Lock user account
[root@security-server ~]# usermod -U analyst2
Unlock user account
[root@security-server ~]# usermod -c "Senior Security Analyst" analyst1
Update user comment/description
‚ö° Account Modification
  • Use -a flag with -G to append, not replace groups
  • Lock compromised accounts immediately with -L
  • Update comments for role changes and tracking
  • Change shells to restrict access: /sbin/nologin
userdel
Remove user accounts from the system with optional home directory removal
[root@security-server ~]# userdel analyst3
Remove user account (keep home directory)
[root@security-server ~]# userdel -r analyst3
Remove user and home directory
[root@security-server ~]# find / -user analyst3 2>/dev/null
Find remaining files owned by deleted user
üóëÔ∏è Safe Account Removal
  • Backup important data before deletion
  • Use find to locate all user-owned files
  • Consider locking instead of deleting for audits
  • Document reason for account removal

groupadd
Create new groups for organizing users and managing permissions
[root@security-server ~]# groupadd security-ops
Create group with next available GID
[root@security-server ~]# groupadd -g 2000 incident-response
Create group with specific GID
[root@security-server ~]# groupadd -r monitoring-svc

üë• Group Organization
  • Use descriptive names: security-ops, incident-response
  • Create groups before adding users for better organization
  • Use system groups (-r) for service accounts
  • Document group purposes for role clarity
groupmod
Modify existing group properties including name and GID
[root@security-server ~]# groupmod -n security-team security-ops
Rename group from security-ops to security-team
[root@security-server ~]# groupmod -g 3000 security-team
Change group GID to 3000
üîÑ Group Modifications
  • Changing GID may affect file ownership
  • Use descriptive names for better administration
  • Test changes in non-production environments
  • Update documentation after modifications
groupdel
Remove groups from the system when no longer needed
[root@security-server ~]# groupdel old-project
Remove unused group
[root@security-server ~]# find / -group old-project 2>/dev/null
Check for files still owned by deleted group
‚ùå Group Removal Safety
  • Cannot delete if it's a user's primary group
  • Find and reassign files before deletion
  • Remove users from group first if needed
  • Document group removal reasoning
newgrp
Temporarily change primary group for file creation in current session
[analyst@security-server ~]$ id
uid=1001(analyst) gid=1001(analyst) groups=1001(analyst),2000(incident-response)
[analyst@security-server ~]$ newgrp incident-response
Switch primary group for this session
[analyst@security-server ~]$ touch incident-report.txt
File will be owned by incident-response group
üîÑ Temporary Group Switch
  • Useful for creating files with specific group ownership
  • Changes only affect current shell session
  • Must be member of target group to switch
  • Primary group reverts after logout
üîë Password & Account Management
‚ñº
passwd
Set and change user passwords with security policy enforcement
[root@security-server ~]# passwd analyst1
Set password for analyst1 user
[analyst@security-server ~]$ passwd
Change own password
[root@security-server ~]# echo "TempPass123!" | passwd --stdin analyst1
Set password non-interactively
üîê Password Security
  • Enforce strong password policies
  • Use temporary passwords for new accounts
  • Force password change on first login
  • Consider password complexity requirements
chage
Configure password and account expiration policies for security compliance
[root@security-server ~]# chage -l analyst1
List account aging information
[root@security-server ~]# chage -M 90 analyst1
Set password to expire in 90 days
[root@security-server ~]# chage -d 0 analyst1
Force password change on next login
[root@security-server ~]# chage -E 2025-12-31 analyst1
Set account to expire on specific date
‚è∞ Account Expiration
  • Set maximum password age for security compliance
  • Use account expiration for temporary access
  • Force password changes for compromised accounts
  • Configure warning periods before expiration
/etc/login.defs
System-wide default settings for user account and password policies
[root@security-server ~]# grep PASS /etc/login.defs
PASS_MAX_DAYS 90
PASS_MIN_DAYS 1
PASS_WARN_AGE 7
[root@security-server ~]# vim /etc/login.defs
Edit default password policies
üìã Default Policies
  • Settings apply to new accounts only
  • Configure UID/GID ranges for different user types
  • Set organization-wide password policies
  • Review and update policies regularly
