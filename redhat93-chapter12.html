Software Packages and RPM

RPM Package File Names

RPM package file names consist of four elements plus the .rpm suffix:

httpd-2.4.37-43.module+el8.5.0.x86_64.rpm

RPM Package Name Components

Name: Package name (httpd)
Version: Software version (2.4.37)
Release: Package release/build (43.module+el8.5.0)
Architecture: Target architecture (x86_64, noarch, etc.)

Examine RPM Packages

The rpm utility is a low-level tool that can retrieve information about the contents of package files and installed packages.

rpm -q package
Query if package is installed
$ rpm -q httpd
httpd-2.4.37-43.module+el8.5.0.x86_64
rpm -qi package
Display detailed package information

Install RPM Packages

rpm -i package.rpm
Install RPM package from local file
# rpm -i /tmp/example-1.0-1.x86_64.rpm
[Package installed successfully]
rpm -ivh package.rpm

# rpm -ivh /tmp/example-1.0-1.x86_64.rpm

Updating / installing...

rpm -U package.rpm

# rpm -U /tmp/example-1.1-1.x86_64.rpm

Updating / installing...
1:example-1.1-1 ################################# [100%]
Cleaning up / removing...

⚠️ Important Note

RPM does not resolve dependencies automatically. Use DNF/YUM for dependency resolution. RPM is primarily used for examining packages and low-level operations.

Manage Software Packages with DNF

Introduction to DNF

DNF (Dandified YUM) is the modern package manager for Red Hat Enterprise Linux. It provides:

  • Dependency Resolution: Automatically resolves package dependencies
  • Repository Management: Manages software repositories
  • Transaction Safety: Safe package transactions with rollback capability
  • Module Streams: Support for application streams and modules
  • Group Operations: Install/remove software groups

RPM Configuration Packages for Local Repositories

rpm -ivh repo-package.rpm
Install repository configuration package
# rpm -ivh https://dl.fedoraproject.org/pub/epel/epel-release-latest-8.noarch.rpm
Retrieving https://dl.fedoraproject.org/pub/epel/epel-release-latest-8.noarch.rpm
Preparing... ################################# [100%]
Updating / installing...
1:epel-release-8-11.el8 ################################# [100%]
cat /etc/yum.repos.d/repofile.repo
View repository configuration
# cat /etc/yum.repos.d/epel.repo
[epel]
name=Extra Packages for Enterprise Linux 8 - $basearch
metalink=https://mirrors.fedoraproject.org/metalink?repo=epel-8&arch=$basearch
enabled=1
gpgcheck=1
countme=1
gpgkey=file:///etc/pki/rpm-gpg/RPM-GPG-KEY-EPEL-8

[epel-debuginfo]
name=Extra Packages for Enterprise Linux 8 - $basearch - Debug
metalink=https://mirrors.fedoraproject.org/metalink?repo=epel-debug-8&arch=$basearch
enabled=0
gpgcheck=1
countme=1
gpgkey=file:///etc/pki/rpm-gpg/RPM-GPG-KEY-EPEL-8

Repository Configuration File Elements

[repo-id]: Repository identifier
name: Human-readable repository name
baseurl/metalink: Repository URL
enabled: Enable/disable repository (1/0)
gpgcheck: Enable GPG signature verification
gpgkey: Path to GPG key file
