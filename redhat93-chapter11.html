Network Models

TCP/IP Network Model

The TCP/IP model is a four-layer networking framework that describes how data should be packaged, transmitted, routed, and received.

Layer Function Protocols
Application User interface and network processes HTTP, HTTPS, FTP, SSH, DNS
Transport Reliable data transfer TCP, UDP
Internet Routing and addressing IP, ICMP, ARP
Network Access Physical transmission Ethernet, Wi-Fi

OSI Model

The OSI model is a seven-layer conceptual framework for network communications.

Layer Name Function
7 Application Network services to applications
6 Presentation Data formatting and encryption
5 Session Manages sessions between applications
4 Transport Reliable data transfer
3 Network Routing and logical addressing
2 Data Link Error detection and frame synchronization
1 Physical Physical transmission of raw bits

IPv4 Networking

IPv4 Networks

IPv4 networks use 32-bit addresses divided into network and host portions. Common private network ranges include:

  • 10.0.0.0/8 (Class A)
  • 172.16.0.0/12 (Class B)
  • 192.168.0.0/16 (Class C)

IPv4 Addresses

IPv4 addresses are written in dotted decimal notation (e.g., 192.168.1.1).

ip addr show
1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 qdisc noqueue state UNKNOWN inet 127.0.0.1/8 scope host lo 2: enp0s3: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc fq_codel state UP inet 192.168.1.100/24 brd 192.168.1.255 scope global dynamic enp0s3

IPv4 Subnets and Netmasks

Subnetting divides networks into smaller segments. Common subnet masks:

CIDR Subnet Mask Hosts
/24 255.255.255.0 254
/25 255.255.255.128 126
/26 255.255.255.192 62

IPv6 Networking

IPv6 Networks

IPv6 uses 128-bit addresses, providing virtually unlimited address space.

IPv6 Addresses

IPv6 addresses use hexadecimal notation separated by colons (e.g., 2001:db8::1).

IPv6 Subnets

IPv6 typically uses /64 subnets for end-user networks, providing 2^64 host addresses per subnet.

ip -6 addr show
2: enp0s3: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 state UP inet6 2001:db8:1::100/64 scope global inet6 fe80::a00:27ff:fe4e:66a1/64 scope link

The ip Command

View Device and Address Information

The ip command is the modern tool for network configuration in Linux.

ip addr show
1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 qdisc noqueue state UNKNOWN link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00 inet 127.0.0.1/8 scope host lo inet6 ::1/128 scope host 2: enp0s3: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc fq_codel state UP link/ether 08:00:27:4e:66:a1 brd ff:ff:ff:ff:ff:ff inet 192.168.1.100/24 brd 192.168.1.255 scope global dynamic enp0s3 inet6 fe80::a00:27ff:fe4e:66a1/64 scope link

Add the -br Option

The -br option provides a brief, readable output format.

ip -br addr show
lo UNKNOWN 127.0.0.1/8 ::1/128 enp0s3 UP 192.168.1.100/24 fe80::a00:27ff:fe4e:66a1/64

The ip link Command

Lists all available network interfaces on your system.

ip link show
1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 qdisc noqueue state UNKNOWN link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00 2: enp0s3: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc fq_codel state UP link/ether 08:00:27:4e:66:a1 brd ff:ff:ff:ff:ff:ff

Network Connectivity and Routing

Verify Connectivity Between Hosts

Use ping to test connectivity to remote hosts.

ping -c 4 8.8.8.8
PING 8.8.8.8 (8.8.8.8) 56(84) bytes of data. 64 bytes from 8.8.8.8: icmp_seq=1 ttl=118 time=15.3 ms 64 bytes from 8.8.8.8: icmp_seq=2 ttl=118 time=14.8 ms 64 bytes from 8.8.8.8: icmp_seq=3 ttl=118 time=15.1 ms 64 bytes from 8.8.8.8: icmp_seq=4 ttl=118 time=14.9 ms --- 8.8.8.8 ping statistics --- 4 packets transmitted, 4 received, 0% packet loss, time 3004ms

Show Routing Information

Use the ip route command to display the routing table.

ip route show
default via 192.168.1.1 dev enp0s3 proto dhcp metric 100 192.168.1.0/24 dev enp0s3 proto kernel scope link src 192.168.1.100 metric 100

NetworkManager Configuration with nmcli

Device Status

The nmcli device status command shows all network devices.

nmcli device status
DEVICE TYPE STATE CONNECTION enp0s3 ethernet connected Wired connection 1 lo loopback unmanaged --

Show Connections

The nmcli connection show command displays all connections.

nmcli connection show
NAME UUID TYPE DEVICE Wired connection 1 5fb06bd0-0bb0-4111-8451-d4e6b0287656 ethernet enp0s3

Add Network Connection

Use nmcli connection add to create new connections.

nmcli connection add type ethernet con-name "Office LAN" ifname enp0s3 ipv4.addresses 192.168.1.50/24 ipv4.gateway 192.168.1.1 ipv4.dns 8.8.8.8
Connection 'Office LAN' (8b5c8a1a-8f93-4828-9e58-4e9b0e3ba718) successfully added.

Activate Connection

The nmcli connection up command activates a network connection.

nmcli connection up "Office LAN"
Connection successfully activated (D-Bus active path: /org/freedesktop/NetworkManager/ActiveConnection/2)

Show Connection Details

View detailed connection information.

nmcli connection show "Office LAN"
connection.id: Office LAN connection.uuid: 8b5c8a1a-8f93-4828-9e58-4e9b0e3ba718 connection.type: 802-3-ethernet connection.interface-name: enp0s3 ipv4.method: manual ipv4.addresses: 192.168.1.50/24 ipv4.gateway: 192.168.1.1 ipv4.dns: 8.8.8.8

Modify Connection

Use nmcli connection modify to update settings.

nmcli connection modify "Office LAN" ipv4.dns 8.8.8.8,8.8.4.4

Delete Connection

Remove a connection from the system.

nmcli connection delete "Office LAN"
Connection 'Office LAN' (8b5c8a1a-8f93-4828-9e58-4e9b0e3ba718) successfully deleted.

Reload Connections

Reload connection files from disk.

nmcli con reload
Note: The /etc/NetworkManager/system-connections/ directory stores connection configurations. Changes made with nmcli con mod are saved here automatically.

Hostname and Name Resolution

The hostname Command

Display or set the system hostname.

hostname
linux-server.cod.edu
hostnamectl set-hostname webserver.cod.edu

Configure Name Resolution

The stub resolver converts hostnames to IP addresses. Configuration is in /etc/nsswitch.conf.

cat /etc/nsswitch.conf | grep hosts
hosts: files dns myhostname
Resolution Order:
  • files: Check /etc/hosts first
  • dns: Query DNS servers from /etc/resolv.conf
  • myhostname: Resolve local hostname

Test DNS Server Connectivity

Use the dig command to test DNS resolution.

dig cod.edu
; <<>> DiG 9.16.1 <<>> cod.edu ;; QUESTION SECTION: ;cod.edu. IN A ;; ANSWER SECTION: cod.edu. 300 IN A 192.203.196.147 ;; Query time: 23 msec ;; SERVER: 8.8.8.8#53(8.8.8.8) ;; MSG SIZE rcvd: 52
