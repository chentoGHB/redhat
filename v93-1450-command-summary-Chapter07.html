Master File Permissions & Security

Learn Linux file permissions through practical scenarios. Understand who can read, write, and execute files, and how to keep your system secure with proper access controls!

Read (r)

r-- = 4
View file contents or list directory

Write (w)

-w- = 2
Modify file or add/remove files in directory

Execute (x)

--x = 1
Run file as program or enter directory

Common Combos

rwx = 7, r-x = 5, r-- = 4
Add numbers: read(4) + write(2) + execute(1)
üëÄ Check What Permissions You Have
You found some files on the system and want to know who can access them. Let's learn to read the permission codes and understand what they mean for security.
1
See All File Details
Use the long listing format to see file permissions, ownership, and other details.
[admin@computer ~]$ ls -la
Shows all files with permissions, owner, group, and dates
2
Check a Specific File
Look at the permissions on a particular file you're interested in.
[admin@computer ~]$ ls -l important-file.txt
Shows detailed info about just one specific file
3
Check Folder Permissions
See the permissions on a directory itself, not the files inside it.
[admin@computer ~]$ ls -ld project-folder
Shows directory permissions (d = directory)
4
Understand the Permission Code
Learn to read the permission string like "-rw-r--r--" to understand access levels.
Example: -rw-r--r-- 1 admin staff 1024 Dec 20 10:30 file.txt
Owner: rw- (read+write), Group: r-- (read only), Others: r-- (read only)
üí° What You Just Did
  • Used "ls -l" to see detailed file information including permissions
  • Used "ls -ld" to check directory permissions specifically
  • Learned to read permission strings: first 3 for owner, next 3 for group, last 3 for others
  • Identified file types: "-" for files, "d" for directories
üõ°Ô∏è Protect Sensitive Files
You have files with passwords and secret information that only you should be able to read. Let's make these files completely private so no one else can access them.
1
Create a Secret File
Make a file with sensitive information that needs protection.
[admin@computer ~]$ passwords.txt')">echo "Admin password: secret123" > passwords.txt
Creates a file containing sensitive password information
2
Check Current Permissions
See what permissions the file currently has - it's probably readable by everyone!
[admin@computer ~]$ ls -l passwords.txt
Shows current permissions - probably -rw-r--r-- (others can read!)
3
Make File Completely Private
Remove all permissions for group and others, leaving only owner access.
[admin@computer ~]$ chmod 600 passwords.txt
Sets permissions to rw------- (only owner can read/write)
Or use symbolic: chmod go-rwx passwords.txt
Removes all permissions for group (g) and others (o)
4
Verify Security
Check that your file is now properly protected.
[admin@computer ~]$ ls -l passwords.txt
Should now show -rw------- (only you can access)
üí° What You Just Did
  • Used "chmod 600" to make files private (read/write for owner only)
  • Used "chmod go-rwx" as an alternative (remove all access for group and others)
  • Protected sensitive files from being read by other users
  • 600 means: owner=6(4+2=read+write), group=0(none), others=0(none)
‚ö° Make Scripts Executable
You wrote a script to automate some security tasks, but when you try to run it, you get "Permission denied". You need to give the script execute permission so it can run.
1
Create a Simple Script
Make a script file that will help with daily security checks.
[admin@computer ~]$ security-check.sh')">echo "#!/bin/bash\necho Checking system security...\nwho\ndate" > security-check.sh
Creates a script that checks who's logged in and the current time
2
Try to Run the Script
Attempt to run the script and see what happens (it will fail!).
[admin@computer ~]$ ./security-check.sh
Will show "Permission denied" because script isn't executable
[admin@computer ~]$ ls -l security-check.sh
Shows permissions like -rw-r--r-- (no execute permission)
3
Add Execute Permission
Give the script execute permission so it can run.
[admin@computer ~]$ chmod +x security-check.sh
Adds execute permission for everyone (owner, group, others)
Or be specific: chmod u+x security-check.sh
Adds execute permission for user (owner) only
4
Run Your Script Successfully
Now your script should work!
[admin@computer ~]$ ./security-check.sh
Script runs successfully and shows security info
[admin@computer ~]$ ls -l security-check.sh
Now shows -rwxr-xr-x (script is executable)
üí° What You Just Did
  • Used "chmod +x" to make files executable
  • Used "chmod u+x" to give execute permission to owner only
  • Learned that scripts need execute permission to run
  • Scripts should start with "#!/bin/bash" to tell the system how to run them
üë• Share Files with Your Team
Your security team needs to share files and work together on projects. You need to set up proper group ownership and permissions so team members can collaborate safely.
1
Create a Team Project File
Make a file that your team will work on together.
[admin@computer ~]$ team-project.txt')">echo "Security Team Project Plan" > team-project.txt
Creates a project file for team collaboration
2
Change Group Ownership
Assign the file to your security team group so all members can access it.
[admin@computer ~]$ chgrp security-team team-project.txt
Changes group ownership to security-team
Or with chown: chown :security-team team-project.txt
Alternative way to change group (: means group only)
3
Give Team Write Permission
Allow your team members to edit the file, not just read it.
[admin@computer ~]$ chmod g+w team-project.txt
Adds write permission for the group
Or with numbers: chmod 664 team-project.txt
Owner: rw(6), Group: rw(6), Others: r(4)
4
Verify Team Access
Check that the file is properly set up for team collaboration.
[admin@computer ~]$ ls -l team-project.txt
Should show -rw-rw-r-- with security-team as group
[admin@computer ~]$ groups
Shows which groups you belong to
üí° What You Just Did
  • Used "chgrp" to change group ownership of files
  • Used "chmod g+w" to give write permission to the group
  • Used "chmod 664" for team collaboration (owner+group write, others read)
  • Used "groups" to see what groups you belong to
üîß Set Default Security
Every time you create new files, they get default permissions that might not be secure. Learn to check and change your umask to control what permissions new files get automatically.
1
Check Current Default Settings
See what umask (default permissions) you currently have.
[admin@computer ~]$ umask
Shows current umask (probably 0022 or 0002)
2
Test Current Defaults
Create a test file and see what permissions it gets automatically.
[admin@computer ~]$ touch test-file.txt
Creates a new file with default permissions
[admin@computer ~]$ ls -l test-file.txt
Shows what permissions new files get (probably -rw-r--r--)
3
Set More Secure Defaults
Change umask to make new files more private by default.
[admin@computer ~]$ umask 027
Sets umask to 027 (removes write for group, all access for others)
For very private: umask 077
Very secure: only owner can access new files
4
Test New Default Security
Create another file and see the improved security.
[admin@computer ~]$ touch secure-file.txt
Creates a file with the new secure defaults
[admin@computer ~]$ ls -l secure-file.txt
With umask 027: -rw-r----- (more secure!)
üí° What You Just Did
  • Used "umask" to check and set default file permissions
  • umask 022 = others can read, umask 027 = others can't access
  • umask 077 = only owner can access (most secure)
  • umask settings only last until you log out (make permanent in .bashrc)
‚ú® Special Shared Folder Permissions
Your team needs a shared folder where everyone's files automatically belong to the team group. You'll use special "setgid" permission so all new files get the right group ownership.
1
Create a Shared Team Directory
Make a folder where your team will collaborate.
[admin@computer ~]$ mkdir shared-projects
Creates a directory for team collaboration
2
Set Group Ownership
Make the folder belong to your security team group.
[admin@computer ~]$ chgrp security-team shared-projects
Changes group ownership to security-team
3
Set Special Group Permissions
Apply setgid so new files automatically get the correct group.
[admin@computer ~]$ chmod 2775 shared-projects
2 = setgid, 775 = rwx for owner/group, r-x for others
Alternative way: chmod g+s shared-projects
Sets the setgid bit using symbolic notation
4
Test the Special Permission
Create a file inside the folder and see if it gets the right group automatically.
[admin@computer ~]$ ls -ld shared-projects
Should show drwxrwsr-x (notice the 's' in group execute position)
[admin@computer ~]$ touch shared-projects/team-file.txt
Creates a file inside the special directory
[admin@computer ~]$ ls -l shared-projects/team-file.txt
File should automatically have security-team as its group!
üí° What You Just Did
  • Used "chmod 2775" to set setgid permission (2 = setgid)
  • Used "chmod g+s" as alternative way to set setgid
  • setgid on directories makes new files inherit the directory's group
  • Look for 's' in the group execute position to identify setgid
