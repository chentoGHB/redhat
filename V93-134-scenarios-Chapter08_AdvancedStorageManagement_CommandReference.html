🗄️ Volume Group Strategy for Security:
Evidence Storage: High-capacity, slow growth, read-mostly access
Investigation Workspace: Medium capacity, active read/write, temporary data
Security Operations: High I/O, log storage, monitoring data
Archive Storage: Large capacity, long-term retention, compliance
System Critical: Small, high availability, OS and critical applications
🗄️ VG Security Best Practices
Segregation: Separate VGs by security classification and access needs
Naming Convention: Clear names indicating purpose (evidence_storage, security_ops)
Capacity Planning: Monitor growth patterns and plan expansion
Redundancy: Consider RAID underneath critical VGs

📊 Storage Health Monitoring Script:
#!/bin/bash
# Enterprise Security Storage Health Check
echo "=== Security Storage Infrastructure Health Check ==="
echo "Date: $(date)"
echo

# Check all security-related volume groups
for vg in $(vgs --noheadings -o vg_name | grep -E "(security|evidence|forensic)"); do
    echo "=== Volume Group: $vg ==="
    vgdisplay $vg | grep -E "(VG Size|Free)"
    
    # Check physical volume health
    echo "Physical Volumes:"
    pvs -S vg_name=$vg -o pv_name,pv_size,pv_used,pv_free
    
    # Check logical volume usage
    echo "Logical Volumes:"
    lvs -S vg_name=$vg -o lv_name,lv_size,segtype,lv_attr
    
    # Check for any failed devices
    if pvs -S vg_name=$vg | grep -q "unknown"; then
        echo "WARNING: Failed devices detected in $vg"
    fi
    echo
done

# Check file system usage
echo "=== Security File System Usage ==="
df -h | grep -E "(evidence|security|forensic|investigation)" | while read filesystem; do
    usage=$(echo $filesystem | awk '{print $5}' | sed 's/%//')
    if [ $usage -gt 85 ]; then
        echo "ALERT: $filesystem usage above 85%"
    fi
done
