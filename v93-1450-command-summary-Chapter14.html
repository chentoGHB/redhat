üåê Set Up Web Console for Remote Management
Install and configure Cockpit web console for secure remote server management. Essential for managing multiple servers and accessing systems remotely.
1
Install Web Console
Install the Cockpit web console package on your server.
[admin@server ~]$ sudo dnf install cockpit
Install Cockpit web console package
[admin@server ~]$ rpm -q cockpit
Verify Cockpit is installed
2
Enable and Start Cockpit Service
Activate the web console service so you can connect remotely.
[admin@server ~]$ sudo systemctl enable --now cockpit.socket
Enable and start the web console socket
[admin@server ~]$ systemctl status cockpit.socket
Check if the service is running properly
3
Configure Firewall Access
Open the firewall to allow web console connections on port 9090.
[admin@server ~]$ sudo firewall-cmd --add-service=cockpit --permanent
Allow Cockpit through the firewall permanently
[admin@server ~]$ sudo firewall-cmd --reload
Apply the firewall changes
4
Access Web Console
Connect to the web console and verify everything is working.
[admin@server ~]$ ss -tlnp | grep 9090
Verify port 9090 is listening
Web Browser: https://your-server-ip:9090
Access the web console in your browser
üí° What You Just Did
  • Installed Cockpit for web-based server management
  • Enabled the cockpit.socket service for remote access
  • Opened firewall port 9090 for secure connections
  • Verified the web console is accessible and running

üîß Generate Diagnostic Reports
Create comprehensive diagnostic reports for troubleshooting and support cases. Essential for documenting system issues and working with technical support.
1
Install SOS Report Tool
Install the SOS report utility for generating diagnostic information.
[admin@server ~]$ sudo dnf install sos
Install the SOS diagnostic utility
[admin@server ~]$ rpm -q sos
Verify SOS is installed
2
Generate Diagnostic Report
Create a comprehensive system diagnostic report for support cases.
[admin@server ~]$ sudo sos report
Generate diagnostic report (may take several minutes)
[admin@server ~]$ sudo sos report --case-id=12345
Generate report with support case ID
3
Generate Report via Web Console
Use the web console to create and download diagnostic reports.
Web Console: Diagnostic Reports ‚Üí Run report
Start diagnostic report from web interface
Web Console: Add label ‚Üí Run report ‚Üí Download
Label and download the completed report
4
Clean Sensitive Data
Remove or obfuscate sensitive information before sharing reports.
[admin@server ~]$ ls -l /var/tmp/sosreport*
Find generated report files
[admin@server ~]$ sudo sos clean /var/tmp/sosreport*.tar.xz
Clean sensitive data from report

Gather specific error messages and log entries related to the problem.

Check kernel messages for errors
3
Generate Comprehensive Report
Create detailed diagnostic report for support team analysis.
[admin@server ~]$ sudo sos report --batch
Generate report without interactive prompts
[admin@server ~]$ sudo sos report --upload
Generate and upload report directly to support
4
Prepare Support Case Information
