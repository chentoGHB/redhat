💿 Security Disk Partitioning
Create secure partitions for evidence storage and data isolation
FUNDAMENTAL
💿 SECURE DATA ISOLATION
New forensic workstation needs secure storage partitioning! Separate evidence storage from system files, create isolated investigation spaces, and implement proper data segregation for security compliance.
MBR (Legacy)
• Max 4 primary partitions
• 2 TiB disk limit
• BIOS compatibility
• Limited for modern security
GPT (Modern)
• 128 partitions max
• 8 ZiB disk limit
• UEFI standard
• Preferred for security

# Security mount options:
# ro       - read-only access
# noexec   - prevent execution of binaries
# nosuid   - ignore SUID/SGID bits
# nodev    - ignore device files
# noatime  - don't update access times (performance)

🔄 Swap Security Considerations:
Encrypted Swap: Use LUKS encryption for sensitive environments
Priority Management: Higher priority swap used first
Size Guidelines: 1-2x RAM for workstations, more for forensic analysis
Performance: SSD swap significantly faster than HDD
Security Risk: Sensitive data may be written to swap files
🔄 Swap Security Best Practices
Encrypted Swap: Always use encryption in security environments
Dedicated Partition: Separate swap from data partitions
Regular Monitoring: Watch swap usage patterns
Secure Disposal: Properly wipe swap partitions when decommissioning
