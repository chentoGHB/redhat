üéØ Permission Bit Values:
Read (r) = 4: View file contents or list directory
Write (w) = 2: Modify file contents or create/delete files

‚ö†Ô∏è SUID/SGID Security Warnings:
Privilege Escalation Risk: SUID programs run with owner privileges

‚≠ê Special Permission Effects:

Combined Usage: Can combine multiple special bits (e.g., 6755 = SUID+SGID)

üé≠ Umask & Default Permissions

Control default permissions for newly created files and directories using umask for consistent security policies.

Umask Commands Default Permissions
$ umask
$ umask 022
$ umask 077
üßÆ Umask Calculator
Files (666 - umask):
666 - 022 = 644 (rw-r--r--)
Directories (777 - umask):
777 - 022 = 755 (rwxr-xr-x)
üé≠ Umask Calculation Process:
File Defaults: Start with 666 (rw-rw-rw-)
Directory Defaults: Start with 777 (rwxrwxrwx)
Subtraction: Default permissions - umask = final permissions
Common Umasks: 022 (standard), 002 (group writable), 077 (user only)
System vs User: System default often 022, can be customized per user
üéØ Umask Security Strategies:
Conservative (077): User-only access for sensitive environments
Standard (022): Owner write, others read - most common
Team (002): Group writable for collaborative work
Public (000): Full permissions - rarely recommended
Service Accounts: Often use 077 for security
‚öôÔ∏è Umask Configuration Workflow:

3

4
Test by creating files and directories
5
Make permanent by adding to ~/.bashrc or system profile

üõ°Ô∏è Enterprise Security Guidelines:

Sensitive File Protection: Use 600 or 640 for critical files
Directory Traversal: Ensure proper directory execute permissions
‚ö†Ô∏è Common Security Vulnerabilities:
World-Writable Files: Files with 666 or 777 permissions
Unnecessary SUID: SUID bits on non-system programs
Weak Directory Permissions: Directories with 777 permissions
Config File Exposure: Readable config files with passwords
Log File Access: Overly permissive log file permissions
üîç Security Audit Process:
1
Scan for world-writable files and directories
2
Identify files with overly permissive permissions
3
Review SUID and SGID files for necessity
4
Audit sensitive directory permissions
5
Document and remediate security issues found
üéØ Security Hardening Recommendations:
System Files: 644 for configs, 600 for sensitive data
Executables: 755 for programs, 750 for admin tools
Log Directories: 750 with appropriate group ownership
Web Content: 644 for files, 755 for directories
Private Keys: 600 with root or service user ownership
