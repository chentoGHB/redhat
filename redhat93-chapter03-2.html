1. Absolute Paths and Relative Paths

Absolute Path: Starts from the root directory (/) and specifies the complete path to a file or directory.
Relative Path: Specifies the path relative to the current working directory.

Examples:

$ pwd /home/student # Absolute path example $ ls /etc/passwd /etc/passwd # Relative path example (from /home/student) $ ls Documents/report.txt Documents/report.txt # Special relative path symbols $ ls . # Current directory $ ls .. # Parent directory $ ls ~ # Home directory

2. Navigate Paths in the File System

Navigation commands allow you to move through the directory structure and view your location.

Examples:

# Change to home directory $ cd ~ # Change to /etc directory $ cd /etc # Go up one directory $ cd .. # Return to previous directory $ cd - /home/student # Print working directory $ pwd /home/student

3. Create Files and Directories

Linux provides various commands to create new files and directories.

Examples:

# Create an empty file $ touch newfile.txt # Create multiple files $ touch file1.txt file2.txt file3.txt # Create a directory $ mkdir myproject # Create nested directories $ mkdir -p projects/web/css # Create multiple directories $ mkdir docs images scripts # Create file with content using echo $ echo "Hello World" > hello.txt

4. Copy Files and Directories

The cp command is used to copy files and directories.

Examples:

# Copy a file $ cp source.txt destination.txt # Copy file to directory $ cp report.txt /home/student/Documents/ # Copy with preservation of attributes $ cp -p original.txt backup.txt # Copy directory recursively $ cp -r mydir/ newdir/ # Copy multiple files to directory $ cp file1.txt file2.txt file3.txt /backup/ # Interactive copy (prompt before overwrite) $ cp -i source.txt dest.txt cp: overwrite 'dest.txt'? y

5. Move Files and Directories

The mv command moves or renames files and directories.

Examples:

# Rename a file $ mv oldname.txt newname.txt # Move file to directory $ mv document.txt Documents/ # Move multiple files $ mv *.txt textfiles/ # Rename directory $ mv olddir/ newdir/ # Interactive move $ mv -i file.txt Documents/ mv: overwrite 'Documents/file.txt'? n # Force move (no prompts) $ mv -f source.txt destination.txt

6. Remove Files and Directories

Warning: The rm command permanently deletes files. Use with caution!

Examples:

# Remove a file $ rm unwanted.txt # Remove multiple files $ rm file1.txt file2.txt file3.txt # Interactive removal $ rm -i important.txt rm: remove regular file 'important.txt'? y # Remove empty directory $ rmdir emptydir/ # Remove directory and contents $ rm -r myproject/ # Force removal of directory $ rm -rf tempdir/ # Remove files matching pattern $ rm *.tmp

7. Create Hard Links

Hard links create additional directory entries for the same file. They share the same inode and data blocks.

Examples:

# Create a hard link $ ln original.txt hardlink.txt # Verify hard links share same inode $ ls -li original.txt hardlink.txt 12345 -rw-r--r-- 2 student student 1024 Jan 10 10:00 hardlink.txt 12345 -rw-r--r-- 2 student student 1024 Jan 10 10:00 original.txt # Check link count $ stat original.txt | grep Links Device: 8,1 Inode: 12345 Links: 2 # Hard links cannot be created for directories $ ln mydir/ dirlink ln: 'mydir/': hard link not allowed for directory

8. Create Symbolic Links

Symbolic (soft) links are special files that point to another file or directory path.

Examples:

# Create symbolic link to file $ ln -s /etc/passwd passwd-link # Create symbolic link to directory $ ln -s /var/log/ log-link # View symbolic links $ ls -l passwd-link lrwxrwxrwx 1 student student 11 Jan 10 10:00 passwd-link -> /etc/passwd # Create relative symbolic link $ ln -s ../Documents/report.txt report-link # Force create symbolic link (overwrite) $ ln -sf newfile.txt existing-link # Find broken symbolic links $ find . -type l ! -exec test -e {} \; -print

9. Match File Names with Shell Expansions

Shell expansions allow pattern matching for file names using wildcards and special characters.

Common Wildcards:

  • * - Matches zero or more characters
  • ? - Matches exactly one character
  • [...] - Matches any one character in the brackets
  • {...} - Brace expansion for multiple options

Examples:

# Match all .txt files $ ls *.txt file1.txt file2.txt report.txt # Match single character $ ls file?.txt file1.txt file2.txt file3.txt # Match character range $ ls file[1-3].txt file1.txt file2.txt file3.txt # Match specific characters $ ls [aeiou]* apple.txt elephant.doc index.html # Brace expansion $ touch file{1..5}.txt $ ls file*.txt file1.txt file2.txt file3.txt file4.txt file5.txt # Multiple extensions $ ls *.{txt,doc,pdf} report.txt notes.doc manual.pdf # Negation $ ls [!0-9]* apple.txt banana.txt config.ini
