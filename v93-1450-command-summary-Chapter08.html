Master Process Monitoring & Control

Learn to see what programs are running, control background tasks, and stop misbehaving processes. Essential skills for keeping your system running smoothly and securely!

Process States

R=Running, S=Sleeping
R: actively using CPU, S: waiting for something

Background Jobs

command &
Run command in background, get prompt back

Job Control

Ctrl+Z, fg, bg
Stop, foreground, background running programs

Kill Signals

TERM=stop, STOP=pause
SIGTERM ends process, SIGSTOP pauses it
üëÅÔ∏è See What Programs Are Running
Your system feels slow and you want to know what programs are currently running. Learn to list all processes and understand what they're doing.
1
List All Running Processes
See all programs running on the system with detailed information.
[admin@computer ~]$ ps aux
Shows all processes with CPU/memory usage and owner
2
See Process Relationships
Show which programs started other programs (parent-child relationships).
[admin@computer ~]$ ps -ef
Shows processes with parent process IDs (PPID)
[admin@computer ~]$ ps --forest
Shows process tree structure visually
3
Find Specific Processes
Look for particular programs you're interested in.
[admin@computer ~]$ ps aux | grep firefox
Finds all processes related to firefox
[admin@computer ~]$ pgrep -l ssh
Finds processes with "ssh" in name and shows names
4
Check Your Own Processes
See only the programs you're running.
[admin@computer ~]$ ps
Shows only your processes in this terminal
[admin@computer ~]$ ps -u $USER
Shows all processes owned by you
üí° What You Just Did
  • Used "ps aux" to see all processes with resource usage
  • Used "ps -ef" to see process relationships (PPID = parent process ID)
  • Used "grep" and "pgrep" to find specific processes
  • Learned that every process has a unique PID (Process ID)
üîÑ Run Tasks in the Background
You have a long-running task like a file backup, but you don't want to wait for it to finish. You want to start it in the background and keep using your terminal.
1
Start a Long Task in Background
Launch a command that will run in the background while you do other things.
[admin@computer ~]$ sleep 300 &
Starts a 5-minute sleep in background (& means background)
[admin@computer ~]$ log-files.txt &')">find /home -name "*.log" > log-files.txt &
Searches for log files in background, saves results to file
2
See Your Background Jobs
Check what tasks you have running in the background.
[admin@computer ~]$ jobs
Lists all background jobs for this terminal session
[admin@computer ~]$ jobs -l
Shows job numbers AND process IDs
3
Bring Background Job to Foreground
If you need to interact with a background job, bring it to the front.
[admin@computer ~]$ fg %1
Brings job #1 to foreground (%1 = job number 1)
[admin@computer ~]$ fg
Brings most recent background job to foreground
4
Send Running Task to Background
If a command is running in foreground and taking too long, send it to background.
[admin@computer ~]$ sleep 300
Start a command in foreground (blocks your terminal)
Press Ctrl+Z: ^Z
Suspends the running command
[admin@computer ~]$ bg %1
Moves suspended job to background and resumes it
üí° What You Just Did
  • Used "&" to start commands in background immediately
  • Used "jobs" to see background jobs and their status
  • Used "fg %1" to bring background job #1 to foreground
  • Used "Ctrl+Z" and "bg" to send running tasks to background
üéÆ Control Running Programs
You have several programs running and need to pause some, resume others, or stop them completely. Learn to control your running programs like a pro.
1
Start Multiple Background Jobs
Create several background tasks so we have something to control.
[admin@computer ~]$ ping google.com > ping.log &
Starts continuous ping, saves output to log file
[admin@computer ~]$ tail -f /var/log/messages &
Monitors system log file in real-time
2
Pause a Running Job
Temporarily stop a job without ending it completely.
[admin@computer ~]$ jobs
See your running background jobs
[admin@computer ~]$ kill -STOP %1
Pauses job #1 (SIGSTOP signal)
3
Resume a Paused Job
Continue a job that was paused.
[admin@computer ~]$ jobs
Check job status (should show "Stopped" for job #1)
[admin@computer ~]$ kill -CONT %1
Resumes job #1 (SIGCONT signal)
Alternative: bg %1
Also resumes job #1 in background
4
Stop Jobs Completely
End jobs when you're done with them.
[admin@computer ~]$ kill -TERM %1
Terminates job #1 politely (SIGTERM signal)
[admin@computer ~]$ kill %2
Terminates job #2 (SIGTERM is default)
üí° What You Just Did
  • Used "kill -STOP" to pause jobs without ending them
  • Used "kill -CONT" or "bg" to resume paused jobs
  • Used "kill -TERM" to end jobs politely
  • Job numbers (like %1, %2) refer to jobs in current terminal session
üõë Stop Misbehaving Programs
A program is using too much CPU or memory, or has frozen and won't respond. You need to forcefully stop it to protect your system.
1
Find the Problem Process
Identify which process is causing problems.
[admin@computer ~]$ ps aux | head -10
Shows top 10 processes sorted by CPU usage
[admin@computer ~]$ ps aux | grep firefox
Look for a specific problematic program
2
Try to Stop It Nicely First
Always try the polite way first - let the program clean up and exit gracefully.
[admin@computer ~]$ kill 1234
Asks process 1234 to terminate nicely (replace with real PID)
[admin@computer ~]$ kill -TERM 1234
Same as above, but explicit (SIGTERM = terminate)
3
Force Stop If Necessary
If the program won't stop nicely, use force (but be careful!).
[admin@computer ~]$ sleep 5; ps aux | grep 1234
Wait a moment, then check if process is still running
[admin@computer ~]$ kill -KILL 1234
Force kill immediately (SIGKILL = no mercy!)
Shorthand: kill -9 1234
Same as SIGKILL (-9 is the number for KILL signal)
4
Verify Process is Gone
Make sure the problem process is actually stopped.
[admin@computer ~]$ ps aux | grep 1234
Check if process 1234 is still running
[admin@computer ~]$ echo $?
Check if kill command succeeded (0 = success)
üí° What You Just Did
  • Always try "kill PID" (SIGTERM) first - gives process chance to clean up
  • Use "kill -9 PID" (SIGKILL) only when process won't respond
  • SIGKILL cannot be ignored - it immediately terminates the process
  • Check process lists to verify the problem process is actually gone
üìä Monitor System Performance
Your system feels slow and you want to see which programs are using the most CPU and memory. Learn to monitor system performance in real-time.
1
Check Overall System Load
See how busy your system is overall.
[admin@computer ~]$ uptime
Shows load average (1min, 5min, 15min) and uptime
[admin@computer ~]$ w
Shows who's logged in and system load
2
Monitor Processes in Real-Time
Watch CPU and memory usage as it changes live.
[admin@computer ~]$ top
Real-time process monitor (press 'q' to quit)
Press in top: P
Sort by CPU usage (highest first)
Press in top: M
Sort by memory usage (highest first)
3
Check CPU and Memory Details
Get more details about your system's resources.
[admin@computer ~]$ lscpu
Shows CPU information (cores, speed, etc.)
[admin@computer ~]$ free -h
Shows memory usage in human-readable format
4
Understand Load Average
Learn what load average numbers mean for your system.
Example output: load average: 0.25, 0.50, 1.20
Last 1 min: light load, 5 min: moderate, 15 min: busy
Rule of thumb: Load < 1.0 = good, Load > 1.0 = busy/overloaded
On multi-core systems, multiply by number of cores
üí° What You Just Did
  • Used "uptime" and "w" to check system load quickly
  • Used "top" for real-time process monitoring with sorting options
  • Used "lscpu" and "free -h" to understand system resources
  • Learned that load average below 1.0 per CPU core is generally good
üéØ Find and Stop Processes by Name
You know a program is causing problems but don't know its process ID. You want to find all instances of a program and stop them quickly.
1
Find Processes by Name
Look for all processes that match a certain name.
[admin@computer ~]$ pgrep firefox
Shows process IDs of all firefox processes
[admin@computer ~]$ pgrep -l firefox
Shows PIDs AND process names
2
Get Detailed Info About Found Processes
See more details about the processes you found.
[admin@computer ~]$ ps aux | grep firefox
Shows detailed info for firefox processes
[admin@computer ~]$ ps -ef | grep firefox
Shows process relationships and full command line
3
Stop All Processes with Same Name
Kill all instances of a program at once.
[admin@computer ~]$ pkill firefox
Terminates ALL firefox processes (SIGTERM)
[admin@computer ~]$ pkill -9 firefox
Force kills ALL firefox processes (SIGKILL)
4
Advanced Process Finding
More precise ways to find and control processes.
[admin@computer ~]$ pgrep -u john firefox
Find firefox processes owned by user 'john' only
[admin@computer ~]$ pkill -STOP firefox
Pause all firefox processes (can resume later)
[admin@computer ~]$ pkill -CONT firefox
Resume all paused firefox processes
üí° What You Just Did
  • Used "pgrep" to find process IDs by name
  • Used "pkill" to terminate all processes with a given name
  • Used "-u username" to target processes by specific user
  • Used different signals with pkill (TERM, KILL, STOP, CONT)
