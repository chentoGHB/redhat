üì° Check Network Information
You need to check what network interfaces are available on your server and see their current IP addresses and status for security monitoring and troubleshooting.
1
List All Network Interfaces
See what network cards and connections are available on your system.
[admin@server ~]$ ip link show
Show all network interfaces (like eth0, ens3, wlan0)
[admin@server ~]$ ip link
Shorter version of the same command
2
Check IP Addresses
See what IP addresses are assigned to your network interfaces.
[admin@server ~]$ ip addr show
Show all IP addresses on all interfaces
[admin@server ~]$ ip addr show eth0
Show IP address for specific interface (eth0)
3
Check Network Statistics
Look for network problems by checking error counts and packet statistics.
[admin@server ~]$ ip -s link show eth0
Show network statistics (packets sent/received, errors)
[admin@server ~]$ ip -br addr
Brief summary of all interfaces and their IPs
4
Check Routing Information
See how your server routes network traffic and what the default gateway is.
[admin@server ~]$ ip route show
Show routing table (where traffic goes)
[admin@server ~]$ ip route | grep default
Show only the default gateway (where unknown traffic goes)
üí° What You Just Did
  • Used "ip link" to see all network interfaces available
  • Used "ip addr" to check what IP addresses are assigned
  • Used "ip -s link" to check for network errors and problems
  • Used "ip route" to see how network traffic is routed
üîó Test Network Connectivity
Your server needs to communicate with other systems. Test whether network connections are working properly and troubleshoot connectivity issues.
1
Test Basic Connectivity
Use ping to test if you can reach other computers or websites.
[admin@server ~]$ ping -c 3 google.com
Send 3 ping packets to google.com
[admin@server ~]$ ping -c 3 192.168.1.1
Test connectivity to your router/gateway
2
Test IPv6 Connectivity
Test IPv6 connections if your network uses IPv6.
[admin@server ~]$ ping6 -c 3 google.com
Test IPv6 connectivity to google.com
[admin@server ~]$ ping6 -c 3 ff02::1%eth0
Find other IPv6 devices on the local network
3
Trace Network Route
See the path your network packets take to reach their destination.
[admin@server ~]$ tracepath google.com
Show route packets take to reach google.com
[admin@server ~]$ tracepath 8.8.8.8
Trace route to Google's DNS server
4
Check Network Services
See what network services are running on your server.
[admin@server ~]$ ss -tuln
Show listening network services (TCP and UDP)
[admin@server ~]$ ss -tuna
Show all network connections (active and listening)
üí° What You Just Did
  • Used "ping -c 3" to send exactly 3 test packets
  • Tested both IPv4 and IPv6 connectivity where available
  • Used "tracepath" to see the network route to destinations
  • Used "ss" to see what network services are running
‚öôÔ∏è Configure Network Connections
You need to set up or change network configuration on your server. Use NetworkManager commands to create and manage network connections.
1
Check Current Network Connections
See what network connections are currently configured and active.
[admin@server ~]$ nmcli con show
Show all configured network connections
[admin@server ~]$ nmcli con show --active
Show only active network connections
2
Check Device Status
See the status of your network devices and what connections they're using.
[admin@server ~]$ nmcli dev status
Show status of all network devices
[admin@server ~]$ nmcli dev show eth0
Show detailed information about eth0 device
3
Create a Static IP Connection
Set up a connection with a fixed IP address instead of DHCP.
[admin@server ~]$ sudo nmcli con add con-name static-eth0 ifname eth0 type ethernet ipv4.method manual ipv4.addresses 192.168.1.100/24 ipv4.gateway 192.168.1.1 ipv4.dns 8.8.8.8
Create static IP connection with IP, gateway, and DNS
[admin@server ~]$ sudo nmcli con up static-eth0
Activate the new static IP connection
4
Manage Connections
Modify, activate, deactivate, or delete network connections as needed.
[admin@server ~]$ sudo nmcli con mod static-eth0 ipv4.dns +8.8.4.4
Add a second DNS server to the connection
[admin@server ~]$ sudo nmcli con down static-eth0
Deactivate the connection
üí° What You Just Did
  • Used "nmcli con show" to list all network connections
  • Used "nmcli dev status" to check network device status
  • Created a static IP connection with specific IP and DNS settings
  • Modified connections with "nmcli con mod" to add DNS servers
üîß Network Troubleshooting
Network connectivity isn't working correctly. Use systematic troubleshooting to identify and fix network problems step by step.
1
Check Physical Network Connection
Make sure your network interface is up and connected to the network.
[admin@server ~]$ ip link show
Check if interfaces show "UP" status
[admin@server ~]$ nmcli dev status
Check if devices show "connected" status
2
Test Local Network
Test if you can reach your gateway (router) and other local devices.
[admin@server ~]$ ip route | grep default
Find your gateway/router IP address
[admin@server ~]$ ping -c 3 $(ip route | grep default | awk '{print $3}' | head -1)
Ping your gateway to test local connectivity
3
Test Internet Connectivity
Check if you can reach the internet and test DNS resolution.
[admin@server ~]$ ping -c 3 8.8.8.8
Test internet connectivity using Google's DNS IP
[admin@server ~]$ ping -c 3 google.com
Test DNS resolution by pinging a hostname
4
Restart Network Services
If problems persist, try restarting network connections and services.
[admin@server ~]$ sudo nmcli con down "System eth0" && sudo nmcli con up "System eth0"
Restart the network connection (use your connection name)
[admin@server ~]$ sudo systemctl restart NetworkManager
Restart NetworkManager service if needed
üí° What You Just Did
  • Checked physical network interface status first
  • Tested local network by pinging the gateway
  • Tested internet by pinging both IP and hostname
  • Restarted network connections when needed
üè∑Ô∏è Hostname and DNS Configuration
Configure your server's hostname and DNS settings so it can resolve domain names and be identified properly on the network.
1
Check Current Hostname
See what your server's current hostname is and check hostname settings.
[admin@server ~]$ hostname
Display current hostname
[admin@server ~]$ hostnamectl status
Show detailed hostname information
2
Set a New Hostname
Change your server's hostname for better identification on the network.
[admin@server ~]$ sudo hostnamectl hostname webserver.company.com
Set permanent hostname (stays after reboot)
[admin@server ~]$ hostname
Verify the new hostname was set
3
Test DNS Resolution
Check if your server can resolve domain names to IP addresses.
[admin@server ~]$ host google.com
Look up IP address for google.com
[admin@server ~]$ dig google.com
Get detailed DNS information for google.com
4
Configure Local Name Resolution
Add local hostname mappings for servers in your network.
[admin@server ~]$ cat /etc/hosts
View current local hostname mappings
[admin@server ~]$ echo "192.168.1.50 database.company.com database" | sudo tee -a /etc/hosts
Add local mapping for database server
[admin@server ~]$ getent hosts database
Test the local hostname mapping
üí° What You Just Did
  • Used "hostnamectl" to check and set permanent hostnames
  • Used "host" and "dig" to test DNS name resolution
  • Added entries to /etc/hosts for local name resolution
  • Used "getent hosts" to test local hostname mappings
üìä Network Monitoring & Performance
Monitor network performance and troubleshoot network problems by checking connection statistics, bandwidth usage, and network service status.
1
Monitor Active Network Connections
See what network connections are currently active on your server.
[admin@server ~]$ ss -tuln
Show listening TCP and UDP services
[admin@server ~]$ ss -tuna | grep ESTAB
Show active established connections
2
Check Network Performance
Monitor network statistics and look for errors or performance issues.
[admin@server ~]$ ip -s link show
Check packet statistics for all interfaces
[admin@server ~]$ watch -n 2 "ip -s link show eth0"
Monitor eth0 statistics in real-time (Ctrl+C to stop)
3
Test Network Speed
Test network latency and connection quality to various destinations.
[admin@server ~]$ ping -c 10 8.8.8.8 | tail -1
Get average ping time to Google DNS
[admin@server ~]$ tracepath -n 8.8.8.8
Show network path with timing information
4
Monitor Network Services
Check which network services are running and accessible.
[admin@server ~]$ ss -tlnp
Show listening TCP services with process names
[admin@server ~]$ nmcli gen permissions
Check NetworkManager permissions and status
üí° What You Just Did
  • Used "ss" to monitor active network connections and services
  • Checked network statistics with "ip -s link" for errors
  • Tested network performance with ping and tracepath
  • Monitored which processes are using network ports
