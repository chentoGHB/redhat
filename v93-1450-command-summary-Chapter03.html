pwd

üîç Cybersecurity Use
  • Essential for maintaining situational awareness during forensic investigations
  • Verify your location before running sensitive commands

cd
Change directory - navigate through the file system hierarchy
[student@workstation ~]$ cd /var/log
[student@workstation log]$ cd ..
[student@workstation var]$ cd ~
üõ°Ô∏è Investigation Shortcuts
  • cd /var/log - Jump to system logs for security analysis
  • cd /etc - Access configuration files
  • cd /tmp - Check temporary files for malware
  • cd .. - Move up one directory level
ls
List directory contents with various options for detailed information
[student@workstation ~]$ ls -la
drwxr-xr-x. 2 student student 4096 Dec 20 14:30 Documents
[student@workstation ~]$ ls -lt
Sort by modification time (newest first)
üîí Security Analysis
  • ls -la shows hidden files and permissions
  • ls -lt reveals recently modified files
  • ls -lh displays human-readable file sizes
  • Check file permissions for security vulnerabilities

üóÇÔ∏è Best Practices
  • Use -p flag to create parent directories automatically
  • Organize incident response data systematically
  • Create separate directories for different evidence types
rmdir
Remove empty directories - safe deletion that prevents accidental data loss

[student@workstation ~]$ rmdir -p old/case/data
‚ö†Ô∏è Safety Features
  • Only removes empty directories - prevents accidental data deletion
  • Use for cleaning up temporary analysis directories
  • Safer alternative to rm -rf for directory removal
üìÑ File Operations
‚ñº
touch
Create empty files or update timestamps
[student@workstation ~]$ touch incident_report.txt
[student@workstation ~]$ touch analysis_{1..5}.log
üìù Forensics Use
  • Create placeholder files for evidence documentation
  • Initialize log files before starting analysis
  • Update file timestamps without modifying content
cp
Copy files and directories - critical for evidence preservation
[student@workstation ~]$ cp suspicious.log evidence/
[student@workstation ~]$ cp -r /var/log /backup/
[student@workstation ~]$ cp -p config.txt config_backup.txt
üîê Evidence Handling
  • Always use -p to preserve timestamps for forensics
  • -r for recursive copying of directories
  • Create working copies to preserve original evidence
mv
Move/rename files and directories
[student@workstation ~]$ mv malware.exe quarantine/
[student@workstation ~]$ mv report.txt final_report.txt
üö® Security Operations
  • Quarantine suspicious files immediately
  • Rename evidence files with case numbers
  • Move sensitive data to secure locations
rm
Remove files and directories - use with extreme caution
[student@workstation ~]$ rm temp_file.log
[student@workstation ~]$ rm -i important_file.txt
rm: remove regular file 'important_file.txt'? y
‚ö†Ô∏è Critical Safety
  • ALWAYS backup evidence before deletion
  • Use -i for interactive confirmation
  • Never use rm -rf / - system destroyer
üîó File Linking
‚ñº
ln
Create hard and symbolic links - essential for evidence preservation
[student@workstation ~]$ ln evidence.log evidence_backup
Creates hard link (same file, two names)
[student@workstation ~]$ ln -s /var/log/secure ~/security_logs
Creates symbolic link (shortcut)
üîê Forensics Applications
  • Hard links protect against accidental deletion
  • Symbolic links provide convenient access paths
  • Use hard links for critical evidence backup
üéØ Pattern Matching & Wildcards
‚ñº
Wildcards
Shell pattern matching for efficient file operations and analysis
[student@workstation ~]$ ls *.log
access.log error.log security.log
[student@workstation ~]$ cp malware* /quarantine/
*
Matches any number of characters
?
Matches exactly one character
[abc]
Matches any one of a, b, or c
[0-9]
Matches any single digit
{log,txt}
Matches either 'log' or 'txt'
~
Represents home directory
üéØ Efficient Analysis
  • Use wildcards to process multiple log files at once
  • Pattern matching speeds up evidence collection
  • Combine with commands: grep "ERROR" *.log

üõ°Ô∏è Security-Critical Directories
  • /var/log/ - System logs, security events, audit trails
  • /etc/ - Configuration files, user accounts, system settings
  • /tmp/ - Often used by malware for temporary files
  • /home/ - User data, personal files, browser caches
  • /usr/bin/ - System binaries that could be modified
