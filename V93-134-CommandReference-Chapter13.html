🐳 Container Lifecycle Management
Create, start, stop, and manage container instances

$podman run registry.redhat.io/rhel9/rhel-guest-image:9.4 echo "Hello World"

Click to copy
$podman run --name webserver -d -p 8080:8080 registry.access.redhat.com/ubi9/httpd-24
Create named container in background with port mapping

$podman run --rm -e NAME="Red Hat" registry.redhat.io/rhel9/rhel-guest-image:9.4 printenv NAME
Run container with environment variable and auto-remove
--rm removes container after exit, -e sets environment variable

$podman ps --format=json

$podman stop webserver

Sends SIGTERM, then SIGKILL after timeout
Click to copy
$podman start webserver
Start stopped container
Restarts with original configuration
Click to copy
$podman restart webserver
Stop and start container
Combination of stop and start operations
Click to copy
$podman rm webserver

Click to copy
$podman rm --force --all

💾 Storage & Volumes
Persistent storage and volume management
Click to copy
$podman run -v /host/path:/container/path:Z image
Mount host directory with SELinux relabeling
:Z relabels for container access (recommended)
Click to copy
$podman run -d --name db01 -e MYSQL_ROOT_PASSWORD=redhat -v /home/user/db_data:/var/lib/mysql:Z registry.lab.example.com/rhel8/mariadb-105
Run database with persistent storage
Data survives container removal
Click to copy
$mkdir /home/user/db_data
Create host directory for volume mounting
Directory must exist before mounting
Click to copy
$podman unshare chown 27:27 /home/user/db_data
Set ownership for rootless container access
Maps container UID/GID to host user namespace
