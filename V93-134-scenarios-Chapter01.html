📜 System Information Script
Create your first security audit script to gather critical system information
BEGINNER
🔍 Real Scenario
As a SOC analyst, you need to quickly gather system information during incident response. Create a script that collects hostname, disk usage, running processes, and network connections.
Click to copy
$vim ~/security_audit.sh
Create your security audit script
Script Content:
#!/usr/bin/bash
# Security Audit Script for Incident Response

echo "=== SECURITY AUDIT REPORT ===" > ~/audit_report.txt
echo "Date: $(date)" >> ~/audit_report.txt
echo "Hostname: $(hostname)" >> ~/audit_report.txt
echo "" >> ~/audit_report.txt

echo "=== SYSTEM INFORMATION ===" >> ~/audit_report.txt
uname -a >> ~/audit_report.txt
echo "" >> ~/audit_report.txt

echo "=== DISK USAGE ===" >> ~/audit_report.txt
df -h >> ~/audit_report.txt
echo "" >> ~/audit_report.txt

echo "=== RUNNING PROCESSES ===" >> ~/audit_report.txt
ps aux --sort=-%cpu | head -20 >> ~/audit_report.txt

$chmod +x ~/security_audit.sh
Make script executable

Click to copy

🛡️ Security Tip
Always use the shebang #!/usr/bin/bash at the start of scripts to ensure they run with the correct interpreter. This prevents security issues from unexpected shell behavior.

for VARIABLE in LIST; do

🔀 Service Status Monitoring
Use if/then statements to monitor critical services and take action
INTERMEDIATE
⚡ Real Scenario
Your firewall service crashed during a security incident. Create a monitoring script that automatically checks if critical security services are running and restarts them if needed.
Click to copy
$if systemctl is-active --quiet firewalld; then
  echo "Firewall is running"
else
  echo "Firewall is DOWN - restarting..."
  sudo systemctl start firewalld
fi
Check firewall status and auto-restart if down
Critical for maintaining security during incidents
Click to copy
$if [ -f /var/log/auth.log ]; then
  grep "Failed password" /var/log/auth.log | tail -10
else
  echo "Auth log not found"
fi
Check for failed login attempts if log exists
-f tests if file exists and is a regular file
Click to copy
$DISK_USAGE=$(df / | grep -v Filesystem | awk "{print $5}" | sed "s/%//")
if [ $DISK_USAGE -gt 90 ]; then
  echo "ALERT: Disk usage is ${DISK_USAGE}%"
fi
Alert if disk usage exceeds 90%
-gt means "greater than" for numeric comparison
🔍 Test Operators:
File tests: -f (file exists), -d (directory), -r (readable)
Numeric tests: -eq (equal), -gt (greater), -lt (less)
String tests: = (equal), != (not equal), -z (empty)
🔍 Log Analysis with Regex
Master pattern matching to find security threats in log files
INTERMEDIATE
🚨 Real Scenario
A security breach has been detected. You need to quickly search through gigabytes of log files to find suspicious IP addresses, failed login attempts, and potential attack patterns.
Click to copy
$grep "^[0-9]\\{1,3\\}\\.[0-9]\\{1,3\\}\\.[0-9]\\{1,3\\}\\.[0-9]\\{1,3\\}" /var/log/auth.log
Find lines starting with IP addresses
^ means start of line, [0-9] matches digits
Click to copy
$grep -i "failed\\|invalid\\|denied" /var/log/auth.log | tail -20
Find authentication failures (case insensitive)
\\| means OR, -i ignores case
Click to copy
$grep "Failed password.*from [0-9]" /var/log/auth.log |
  awk "{print $11}" | sort | uniq -c | sort -nr
Count failed login attempts by IP address
.* matches any characters, shows most frequent first
Click to copy
$grep "$(date +%b\\ %d)" /var/log/syslog |
  grep -E "(error|critical|alert)"
Find today's critical system errors
-E enables extended regex, () groups patterns
📝 Regex Basics:
^ = start of line, $ = end of line
. = any character, * = zero or more
[0-9] = any digit, [a-z] = any lowercase letter
\\| = OR operator, () = grouping
🛡️ Incident Response Automation
Combine all skills for real cybersecurity incident response
ADVANCED
🚨 Real Scenario
Your organization detected a potential security breach. You need a comprehensive script that gathers forensic evidence, checks for indicators of compromise, and documents everything for the incident response team.
Click to copy
$vim ~/incident_response.sh
Create comprehensive incident response script
Complete IR Script:
#!/usr/bin/bash
# Incident Response Evidence Collection Script

INCIDENT_ID="IR-$(date +%Y%m%d-%H%M%S)"
REPORT_DIR="~/incident_reports/$INCIDENT_ID"
mkdir -p "$REPORT_DIR"

echo "Starting incident response for $INCIDENT_ID"

# System snapshot
echo "=== SYSTEM SNAPSHOT ===" > "$REPORT_DIR/system_info.txt"
date >> "$REPORT_DIR/system_info.txt"
uname -a >> "$REPORT_DIR/system_info.txt"
uptime >> "$REPORT_DIR/system_info.txt"

# Check for suspicious processes
ps aux --sort=-%cpu | head -20 > "$REPORT_DIR/top_processes.txt"

# Network connections
netstat -tuln > "$REPORT_DIR/network_connections.txt"

# Recent authentication attempts
if [ -f /var/log/auth.log ]; then
    grep "$(date +%b\\ %d)" /var/log/auth.log | \
    grep -E "(Failed|Invalid|Denied)" > "$REPORT_DIR/auth_failures.txt"
fi

# Check for new files in /tmp (common attack vector)
find /tmp -type f -mtime -1 -exec ls -la {} \\; > "$REPORT_DIR/tmp_files.txt"

echo "Evidence collected in $REPORT_DIR"
Click to copy
$for SUSPICIOUS_IP in $(grep "Failed password" /var/log/auth.log |
  awk "{print $11}" | sort | uniq -c | sort -nr |
  head -5 | awk "{print $2}")
do
  echo "Investigating $SUSPICIOUS_IP"
  whois $SUSPICIOUS_IP | grep -E "(Country|NetName)"
done
Investigate top 5 IPs with failed login attempts
Combines loops, pipes, and regex for threat intelligence
⚠️ Critical Security Note
Always preserve original log files when collecting evidence. Use cp or rsync to create copies before analysis to maintain chain of custody for legal proceedings.

🐛 Common Mistakes:
Missing spaces: if[$var -eq 1] → if [ $var -eq 1 ]
Unquoted variables: echo $var → echo "$var"
Wrong permissions: Use chmod +x script.sh
Path issues: Use ./script.sh in current directory
