üíΩ Check Disk and File System Information
Check disk space usage, identify storage devices, and monitor file system capacity for proper server maintenance and security monitoring.
1
Check Disk Space Usage
Monitor how much disk space is used and available on all mounted file systems.
[admin@server ~]$ df -h
Show disk usage in human-readable format
[admin@server ~]$ df -T
Show disk usage with file system types
2
List Block Devices
See all available storage devices and their partition structure.
[admin@server ~]$ lsblk
List all block devices in tree format
[admin@server ~]$ lsblk -f
Show file system types and mount points
3
Check Detailed Partition Info
Get detailed information about devices including UUIDs and file system details.
[admin@server ~]$ lsblk -fp
Show full paths, UUIDs, and file system info
[admin@server ~]$ blkid
Show UUIDs and labels for all block devices
4
Monitor Disk Usage by Directory
Check which directories are using the most disk space.
[admin@server ~]$ du -sh /var/log
Check size of log directory (important for security)
[admin@server ~]$ du -h --max-depth=1 /var | sort -hr
Find largest directories in /var
üí° What You Just Did
  • Used "df -h" to check disk space usage in readable format
  • Used "lsblk" to see all storage devices and partitions
  • Used "blkid" to get UUIDs for reliable device identification
  • Used "du" to find directories using too much disk space
üîó Mount and Unmount Operations
Connect external drives, USB devices, and additional storage to your server. Essential for accessing backup drives and security evidence storage.
1
Create Mount Points
Prepare directories where you'll attach storage devices.
[admin@server ~]$ sudo mkdir /mnt/backup
Create mount point for backup drive
[admin@server ~]$ sudo mkdir /mnt/evidence
Create mount point for forensic evidence
2
Mount by Device Name
Connect storage devices using their device names.
[admin@server ~]$ sudo mount /dev/sdb1 /mnt/backup
Mount external drive to backup directory
[admin@server ~]$ sudo mount /dev/sdc1 /mnt/evidence
Mount forensic evidence drive
3
Mount by UUID (Recommended)
Use UUIDs for reliable mounting that won't change if devices are added.
[admin@server ~]$ sudo mount UUID="a04c511a-b805-4ec2-981f-42d190fc9a65" /mnt/backup
Mount using UUID (get UUID with lsblk -f)
[admin@server ~]$ mount | grep backup
Verify the backup drive is mounted
4
Safely Unmount Drives
Properly disconnect storage devices to prevent data corruption.
[admin@server ~]$ sudo umount /mnt/backup
Safely unmount backup drive
[admin@server ~]$ sudo umount /dev/sdc1
Unmount evidence drive by device name
üí° What You Just Did
  • Created mount points with "mkdir" in /mnt directory
  • Mounted drives using both device names and UUIDs
  • Used UUIDs for reliable device identification
  • Safely unmounted drives with "umount" to prevent corruption
üîç Find Files by Name and Location
Search for configuration files, security logs, and system files. Essential for security analysis, troubleshooting, and system administration.
1
Find Files by Exact Name
Search for files with specific names across the entire system.
[admin@server ~]$ find /etc -name sshd_config
Find SSH server configuration file
[admin@server ~]$ find /var/log -name "*.log"
Find all log files in /var/log directory
2
Find Files with Wildcards
Use wildcards to find files with partial name matches.
[admin@server ~]$ find /etc -name "*pass*"
Find files containing "pass" in their name
[admin@server ~]$ find /home -name "*.txt"
Find all text files in home directories
3
Case-Insensitive Search
Search for files without worrying about uppercase/lowercase.
[admin@server ~]$ find / -iname "*error*"
Find files with "error" in any case (Error, ERROR, etc.)
[admin@server ~]$ find /var/log -iname "*security*"
Find security-related logs regardless of case
4
Find in Current Directory
Search in the current directory and its subdirectories.
[admin@server ~]$ find . -name "*.conf"
Find configuration files in current directory
[admin@server ~]$ find . -name "backup*"
Find files starting with "backup"
üí° What You Just Did
  • Used "find" with "-name" for exact filename searches
  • Used wildcards (*) to match partial filenames
  • Used "-iname" for case-insensitive searches
  • Used "." to search in the current directory tree
üî¨ Advanced Find Operations
Search files by permissions, ownership, size, and modification time. Critical for security auditing and system troubleshooting.
1
Find by Permissions
Locate files with specific permissions - important for security auditing.
[admin@server ~]$ find /home -perm 777
Find files with dangerous 777 permissions
[admin@server ~]$ find /etc -perm -o=w
Find files writable by others (security risk)
2
Find by Ownership
Search for files owned by specific users or groups.
[admin@server ~]$ find /home -user apache
Find files owned by apache user
[admin@server ~]$ find /var -group wheel
Find files owned by wheel group
3
Find by Size
Locate large files that might be consuming disk space or security logs.
[admin@server ~]$ find /var/log -size +100M
Find log files larger than 100MB
[admin@server ~]$ find /home -size -1k
Find very small files (less than 1KB)
4
Find by Modification Time
Search for recently modified files - useful for security incident analysis.
[admin@server ~]$ find /etc -mmin -60
Find files modified in last 60 minutes
[admin@server ~]$ find /var/log -mmin +1440
Find files older than 24 hours (1440 minutes)
üí° What You Just Did
  • Used "-perm" to find files with specific permissions
  • Used "-user" and "-group" to find files by ownership
  • Used "-size" with +/- to find large or small files
  • Used "-mmin" to find recently modified or old files
‚ö° Locate Files Quickly
Use the locate database for lightning-fast file searches. Perfect for quickly finding configuration files and system files by name.
1
Update Locate Database
Update the file database to include recently created files.
[admin@server ~]$ sudo updatedb
Update the locate database manually
[admin@server ~]$ ls -la /var/lib/mlocate/mlocate.db
Check when database was last updated
2
Basic Locate Searches
Quickly find files by name using the database index.
[admin@server ~]$ locate passwd
Find all files containing "passwd" in name or path
[admin@server ~]$ locate sshd
Find SSH daemon related files quickly
3
Limited and Case-Insensitive Searches
Control search results with options for better output.
[admin@server ~]$ locate -n 5 config
Show only first 5 results for "config"
[admin@server ~]$ locate -i LOG
Case-insensitive search for "log" files
4
Locate vs Find Comparison
Understand when to use locate vs find for different situations.
[admin@server ~]$ time locate httpd.conf
Fast search using database (very quick)
[admin@server ~]$ time find / -name httpd.conf 2>/dev/null
Real-time search (slower but more current)
üí° What You Just Did
  • Used "updatedb" to refresh the locate database
  • Used "locate" for lightning-fast file searches
  • Limited results with "-n" and used "-i" for case-insensitive searches
  • Compared locate (fast/database) vs find (slow/real-time) speeds
üõ†Ô∏è Troubleshoot Mount Issues
Fix common mounting problems like "device busy" errors, permission issues, and file system problems. Essential for maintaining storage access.
1
Check What's Using Mount Point
Find processes that are preventing unmounting with "device busy" errors.
[admin@server ~]$ lsof /mnt/backup
Show processes using the mount point
[admin@server ~]$ fuser -v /mnt/backup
Alternative way to check what's using the mount
2
Force Kill Processes Using Mount
Safely terminate processes blocking the unmount operation.
[admin@server ~]$ fuser -k /mnt/backup
Kill all processes using the mount point
[admin@server ~]$ sudo umount /mnt/backup
Try unmounting again after killing processes
3
Force Unmount (Last Resort)
Force unmounting when normal methods fail - use with caution.
[admin@server ~]$ sudo umount -f /mnt/backup
Force unmount (may cause data loss!)
[admin@server ~]$ sudo umount -l /mnt/backup
Lazy unmount (detach when no longer busy)
4
Check File System Errors
Verify file system integrity if mounting problems persist.
[admin@server ~]$ sudo fsck /dev/sdb1
Check and repair file system (device must be unmounted)
[admin@server ~]$ dmesg | grep -i error | tail -10
Check system messages for storage errors
üí° What You Just Did
  • Used "lsof" and "fuser" to identify blocking processes
  • Used "fuser -k" to kill processes preventing unmount
  • Used force unmount options as emergency measures
  • Used "fsck" to check file system integrity
