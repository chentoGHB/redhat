📝 Configuration File Example:
# /etc/selinux/config
# This file controls the state of SELinux on the system.
SELINUX=enforcing
# SELINUXTYPE= can take one of these three values:
#     targeted - Targeted processes are protected,
#     minimum - Modification of targeted policy. Only selected processes are protected. 
#     mls - Multi Level Security protection.
SELINUXTYPE=targeted

🔒 Enforcement Mode Best Practices
Production Systems: Always use enforcing mode for maximum security
Development/Testing: Use permissive mode to identify policy issues
Emergency Access: Temporary permissive mode for troubleshooting
Never Disable: Disabled mode removes all SELinux protection

⚙️ Key SELinux Booleans for Security:
httpd_enable_homedirs: Allow Apache access to user home directories
httpd_can_network_connect: Allow Apache to make outbound network connections
httpd_use_cifs: Allow Apache to access CIFS/SMB shares
ftp_home_dir: Allow FTP access to user home directories
ssh_sysadm_login: Allow SSH login for system administrators
allow_execmem: Allow programs to execute memory (potential security risk)
⚡ Boolean Security Principles
Least Privilege: Only enable booleans required for functionality
Document Changes: Keep track of which booleans are enabled and why
Regular Review: Periodically audit enabled booleans
Persistent vs Temporary: Use -P for production, temporary for testing

🔍 Troubleshooting Priority Order:
1. File Contexts: Most common issue - wrong contexts on moved/copied files
2. SELinux Booleans: Feature disabled that application requires
3. File Permissions: Traditional permissions may also block access
4. Custom Policy: Last resort - create specific policy rules
Never: Disable SELinux or set to permissive permanently
🔧 Troubleshooting Best Practices
Root Cause Analysis: Fix the underlying issue, not just the symptoms
Security First: Avoid creating overly permissive custom policies
Document Changes: Keep record of all SELinux modifications
Test Thoroughly: Verify fixes don't introduce new vulnerabilities
