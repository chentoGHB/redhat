🔐 SELinux Port Context Error
SELinux is preventing /usr/sbin/httpd from name_bind access on the tcp_socket port 8443. ***** Plugin bind_ports (99.5 confidence) suggests ************************ If you want to allow /usr/sbin/httpd to bind to network port 8443 Then you need to modify the port type. Do # semanage port -a -t PORT_TYPE -p tcp 8443 where PORT_TYPE is one of: http_cache_port_t, http_port_t, jboss_management_port_t
