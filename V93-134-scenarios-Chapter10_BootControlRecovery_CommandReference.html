🔑 Root Password Recovery
Reset lost root password using rd.break for emergency system access
ADVANCED
🔑 EMERGENCY ACCESS RECOVERY
Critical security system lost root access during incident response! Former administrator unavailable and root password unknown. Use rd.break to interrupt boot process, access initramfs shell, mount real filesystem, and reset root password for emergency system recovery.
1
Boot system and interrupt GRUB2 bootloader menu
2
Edit rescue kernel entry and append rd.break parameter
3
Access emergency shell in initramfs environment
4
Mount real filesystem and change root password
5
Fix SELinux context and continue normal boot
🔑 GRUB2 rd.break Configuration:
# Boot process: 1. Restart system 2. Press any key to interrupt GRUB countdown 3. Select rescue kernel (has 'rescue' in name) 4. Press 'e' to edit selected entry 5. Find line starting with 'linux' 6. Navigate to end of line (press End key) 7. Append: rd.break 8. Press Ctrl+x to boot with changes # Example modified kernel line: linux /vmlinuz-5.14... root=/dev/vda4 rd.break # This breaks into initramfs just before mounting real root

🔧 Common fstab Boot Failures:
Invalid UUID: Device UUID changed after disk replacement or partition modification
Missing Mount Point: Mount directory doesn't exist or was deleted
Wrong Device Name: /dev/sdb changed to /dev/sdc after hardware changes
Corrupted Filesystem: Filesystem corruption prevents mounting
Network Mounts: NFS server unavailable, causing boot timeout
🔧 Filesystem Recovery Best Practices
Use nofail Option: Add nofail to non-critical mounts in fstab
UUID Identification: Always use UUIDs instead of device names
Backup fstab: Keep backup copy before making changes
Test Before Reboot: Always test mount -a before rebooting
