🔧 Manual Mounting Security Practices
Emergency Access: Fast temporary mounting for incident response
Sync Option: Use sync for critical data to ensure immediate writes
Read-Only: Mount sensitive feeds as read-only when appropriate
Clean Unmount: Always unmount cleanly to prevent data corruption
🔧 NFS Mount Options for Security:
rw/ro: Read/write or read-only access control
sync: Synchronous writes for data integrity
sec=krb5: Kerberos authentication for secure access
vers=4: Force NFSv4 for enhanced security features
soft/hard: Timeout behavior for network issues

#!/bin/bash # Enterprise Security Storage Infrastructure Health Check echo "=== Enterprise Security Storage Health Check ===" echo "Date: $(date)" echo "Hostname: $(hostname)" echo # Check all NFS mounts echo "Active NFS Mounts:" mount | grep nfs4 | while read line; do mountpoint=$(echo $line | awk '{print $3}') server=$(echo $line | awk '{print $1}') echo " Mount: $mountpoint" echo " Server: $server" df -h $mountpoint | tail -1 echo done # Check automounter status echo "Automounter Service Status:" systemctl is-active autofs systemctl is-enabled autofs # Check for failed mounts echo "Failed Mount Attempts:" journalctl -u autofs --since "1 hour ago" | grep -i error || echo "No errors found" # Storage utilization alerts echo "Storage Utilization Alerts (>85% usage):" df -h | grep -E "/security|/evidence|/intel" | awk '$5+0 > 85 {print "ALERT: " $6 " is " $5 " full"}' # Network connectivity test echo "Storage Server Connectivity:" for server in evidence-primary evidence-backup intel-feeds log-storage; do if ping -c 1 $server.corp.com &>/dev/null; then echo " $server.corp.com: Online" else echo " $server.corp.com: OFFLINE" fi done
