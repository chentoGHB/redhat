üëÅÔ∏è Check What Services Are Running
You need to see what services are currently running on your system, which ones are enabled at boot, and check the status of important services like SSH and firewall.
1
List All Running Services
See all services that are currently active on your system.
[admin@computer ~]$ systemctl list-units --type=service
Shows all loaded and active services
2
Check Specific Service Status
Get detailed information about a specific service.
[admin@computer ~]$ systemctl status sshd
Shows detailed status of SSH service
[admin@computer ~]$ systemctl status firewalld
Checks firewall service status
3
Quick Service State Checks
Quickly check if services are active or enabled without detailed output.
[admin@computer ~]$ systemctl is-active sshd
Returns "active" or "inactive" for SSH service
[admin@computer ~]$ systemctl is-enabled sshd
Returns "enabled" or "disabled" for auto-start at boot
4
Find Failed Services
Identify services that failed to start properly.
[admin@computer ~]$ systemctl --failed --type=service
Lists all services that failed during startup
[admin@computer ~]$ systemctl is-failed chronyd
Checks if a specific service failed
üí° What You Just Did
  • Used "systemctl list-units --type=service" to see all running services
  • Used "systemctl status" to get detailed information about specific services
  • Used "is-active" and "is-enabled" for quick status checks
  • Used "--failed" to find services that didn't start properly

üéÆ Start, Stop, and Restart Services
A service has crashed or you need to apply configuration changes. You need to restart services, stop unnecessary services, and reload configurations without downtime.
1
Start a Stopped Service
Start a service that isn't currently running.
[admin@computer ~]$ sudo systemctl start httpd
Starts the Apache web server service
[admin@computer ~]$ systemctl status httpd
Verify the service started successfully
2
Stop a Running Service
Stop a service that you no longer need or that's causing problems.
[admin@computer ~]$ sudo systemctl stop httpd
Stops the Apache web server service
[admin@computer ~]$ systemctl is-active httpd
Confirm the service is now inactive
3
Restart a Service
Restart a service to apply configuration changes or fix problems.
[admin@computer ~]$ sudo systemctl restart sshd
Stops and then starts SSH service (new PID)
[admin@computer ~]$ sudo systemctl reload sshd
Reloads config without stopping service (same PID)
4
Safe Restart Options
Use safe options when you're not sure if a service supports reload.
[admin@computer ~]$ sudo systemctl reload-or-restart httpd
Tries reload first, restarts if reload not supported
[admin@computer ~]$ sudo systemctl try-restart sshd
Only restarts if service is already running
üí° What You Just Did
  • Used "start" and "stop" to manually control service states
  • Used "restart" to completely stop and start services (new process ID)
  • Used "reload" to update configuration without downtime
  • Used safe options like "reload-or-restart" and "try-restart"
üöÄ Control Auto-Start at Boot
You want to control which services start automatically when the system boots. Some services should always start, others should only be started manually when needed.
1
Enable Service to Start at Boot
Make sure critical services start automatically when the system reboots.
[admin@computer ~]$ sudo systemctl enable firewalld
Enable firewall to start automatically at boot
[admin@computer ~]$ sudo systemctl enable --now sshd
Enable and immediately start SSH service
2
Disable Auto-Start
Prevent services from starting automatically to save resources or for security.
[admin@computer ~]$ sudo systemctl disable cups
Disable printing service auto-start (start manually when needed)
[admin@computer ~]$ sudo systemctl disable --now httpd
Disable and immediately stop web server
3
Check Auto-Start Status
Verify which services are enabled or disabled for auto-start.
[admin@computer ~]$ systemctl is-enabled firewalld
Check if firewall will start at boot
[admin@computer ~]$ systemctl list-unit-files --type=service | grep enabled
List all services enabled for auto-start
4
Mask Unwanted Services
Completely prevent services from being started, even manually.
[admin@computer ~]$ sudo systemctl mask sendmail
Completely block sendmail service (if you use postfix instead)
[admin@computer ~]$ sudo systemctl unmask sendmail
Remove mask to allow service to be started again
üí° What You Just Did
  • Used "enable" to make services start automatically at boot time
  • Used "disable" to prevent auto-start but still allow manual start
  • Used "--now" to enable/disable and start/stop in one command
  • Used "mask" to completely prevent a service from starting
üîß Troubleshoot Service Problems
A service won't start or keeps crashing. You need to investigate what's going wrong, check logs for error messages, and fix the problem.
1
Check Service Status in Detail
Get detailed information about a problematic service.
[admin@computer ~]$ systemctl status httpd -l
Shows full status with complete log lines
[admin@computer ~]$ systemctl status httpd --no-pager
Shows status without paging (good for scripts)
2
Check Service Logs
Look at recent log entries to find error messages and clues.
[admin@computer ~]$ journalctl -u httpd
Shows all log entries for httpd service
[admin@computer ~]$ journalctl -u httpd --since today
Shows only today's log entries for httpd
3
Follow Logs in Real-Time
Watch log messages as they happen while trying to start the service.
[admin@computer ~]$ journalctl -u httpd -f
Follow httpd logs in real-time (like tail -f)
[admin@computer ~]$ journalctl -u httpd -n 50
Shows last 50 log entries for httpd
4
Test Configuration
Test service configuration for syntax errors before starting.
[admin@computer ~]$ httpd -t
Test Apache configuration for syntax errors
[admin@computer ~]$ nginx -t
Test Nginx configuration for syntax errors
[admin@computer ~]$ sshd -t
Test SSH configuration for syntax errors
üí° What You Just Did
  • Used "systemctl status -l" to get full detailed service information
  • Used "journalctl -u service" to read service-specific log entries
  • Used "-f" flag to follow logs in real-time for troubleshooting
  • Tested service configurations before starting to catch errors early
üîó Understanding Service Dependencies
Some services depend on others to work properly. You need to understand these relationships to troubleshoot startup problems and ensure services start in the right order.
1
List Service Dependencies
See what services a specific service depends on to function.
[admin@computer ~]$ systemctl list-dependencies sshd
Shows what SSH service depends on to start
[admin@computer ~]$ systemctl list-dependencies httpd --all
Shows all dependencies (including recursive ones)
2
Find Reverse Dependencies
See which services depend on a particular service.
[admin@computer ~]$ systemctl list-dependencies --reverse network.target
Shows services that depend on networking
[admin@computer ~]$ systemctl list-dependencies --reverse sshd
Shows what depends on SSH service
3
Handle Socket Dependencies
Understand services that are started by socket activation.
[admin@computer ~]$ systemctl list-units --type=socket
Shows all active socket units
[admin@computer ~]$ systemctl stop cups.service
Stopping a service might show dependency warnings
4
Check Targets and Runlevels
Understand system targets (equivalent to old runlevels).
[admin@computer ~]$ systemctl get-default
Shows current default target (usually multi-user.target)
[admin@computer ~]$ systemctl list-dependencies multi-user.target
Shows all services started in multi-user mode
üí° What You Just Did
  • Used "list-dependencies" to understand service startup requirements
  • Used "--reverse" to find which services depend on a specific service
  • Learned about socket-activated services and their relationships
  • Understood system targets and how services are grouped
