‚öôÔ∏è Copy Options & Flags:

cp -p file dest: Preserve permissions, ownership, and timestamps
cp -i file dest: Interactive mode - prompts before overwriting
cp -u file dest: Update mode - copy only if source is newer

‚ö° Move vs Copy:
Move (mv): Transfers file to new location, removes from original location
Copy (cp): Creates duplicate, original file remains unchanged
Efficiency: mv is faster for same filesystem (just updates directory entry)
Safety: mv is atomic operation - file is never duplicated
Cross-filesystem: mv automatically copies then deletes when crossing filesystems

üéØ Link Types Explained:
Hard Links (ln): Multiple filenames pointing to same inode/data
Symbolic Links (ln -s): File containing path to another file
Inode Sharing: Hard links share storage, symbolic links are separate files
Cross-filesystem: Symbolic links work across filesystems, hard links don't
Directory Links: Only symbolic links can point to directories
üéØ Security & Admin Applications:
Log Access: Create convenient links to frequently accessed log files
Tool Organization: Link security tools to standard PATH directories
Configuration Management: Link active configs to version-controlled copies
Backup Strategies: Hard links for space-efficient incremental backups
System Integration: Link custom scripts to system directories
