üîç Find and Search for Software
You need to find security tools or applications to install on your server. Search for available software packages using different methods.
1
Search by Package Name
Find packages by searching their names or descriptions.
[admin@server ~]$ dnf search nmap
Search for network scanning tool (nmap)
[admin@server ~]$ dnf search "security scanner"
Search for security scanning tools by description
2
List Available Packages
Browse available software packages and see what's already installed.
[admin@server ~]$ dnf list available | grep security
List available security-related packages
[admin@server ~]$ dnf list installed
List all currently installed packages
3
Get Package Information
Get detailed information about a package before installing it.
[admin@server ~]$ dnf info nmap
Show detailed information about nmap package
[admin@server ~]$ dnf provides /usr/bin/nmap
Find which package provides the nmap command
4
Check Package Dependencies
See what other packages will be installed as dependencies.
[admin@server ~]$ dnf deplist nmap
Show what dependencies nmap needs
[admin@server ~]$ dnf repoquery --requires nmap
List what packages nmap requires to work
üí° What You Just Did
  • Used "dnf search" to find packages by name or description
  • Used "dnf list" to see available and installed packages
  • Used "dnf info" to get detailed package information before installing
  • Used "dnf deplist" to check what dependencies are required
‚¨áÔ∏è Install Software Packages
Install security tools, text editors, and other software your server needs for administration and security operations.
1
Install Individual Packages
Install specific software packages and their dependencies.
[admin@server ~]$ sudo dnf install nmap
Install nmap network scanning tool
[admin@server ~]$ sudo dnf install vim-enhanced
Install the full version of vim text editor
2
Install Multiple Packages
Install several packages in one command to save time.
[admin@server ~]$ sudo dnf install git wget curl
Install git, wget, and curl tools together
[admin@server ~]$ sudo dnf install tcpdump wireshark-cli
Install network monitoring tools
3
Install with Automatic Yes
Skip confirmation prompts for automation and scripts.
[admin@server ~]$ sudo dnf install -y htop
Install htop without asking for confirmation
[admin@server ~]$ sudo dnf install -y tree unzip
Install utility tools automatically
4
Verify Installation
Check that software was installed correctly and is working.
[admin@server ~]$ which nmap
Check if nmap command is available
[admin@server ~]$ rpm -q nmap
Verify nmap package is installed with version
üí° What You Just Did
  • Used "sudo dnf install" to install packages with admin privileges
  • Installed multiple packages in one command to save time
  • Used "-y" flag to automatically answer "yes" to prompts
  • Verified installation with "which" and "rpm -q" commands
üîÑ Update System and Packages
Keep your system secure by regularly updating packages and installing security patches. This is critical for cybersecurity.
1
Check Available Updates
See what updates are available before installing them.
[admin@server ~]$ dnf check-update
List all packages that have updates available
[admin@server ~]$ dnf check-update --security
Check only for security updates (critical for cybersecurity)
2
Install Security Updates
Install critical security patches first to protect your system.
[admin@server ~]$ sudo dnf update --security
Install only security updates (recommended for production)
[admin@server ~]$ sudo dnf update --security -y
Install security updates automatically
3
Update All Packages
Update all installed packages to their latest versions.
[admin@server ~]$ sudo dnf update
Update all packages on the system
[admin@server ~]$ sudo dnf update vim
Update only a specific package (vim)
4
Clean Package Cache
Clean up downloaded package files to save disk space.
[admin@server ~]$ sudo dnf clean packages
Remove cached package files
[admin@server ~]$ sudo dnf clean all
Clean all cache data (packages, metadata, etc.)
üí° What You Just Did
  • Used "dnf check-update" to see available updates before installing
  • Used "--security" flag to focus on critical security patches
  • Updated all packages with "dnf update" for system maintenance
  • Cleaned package cache to free up disk space after updates
üóëÔ∏è Remove Software Packages
Remove unnecessary software and packages you no longer need to keep your system clean and reduce attack surface.
1
Remove Individual Packages
Remove specific packages you no longer need.
[admin@server ~]$ sudo dnf remove telnet
Remove insecure telnet package
[admin@server ~]$ sudo dnf remove -y old-package
Remove package without confirmation prompt
2
Check Dependencies Before Removing
See what other packages depend on the one you want to remove.
[admin@server ~]$ dnf repoquery --whatrequires httpd
Check what packages need httpd before removing
[admin@server ~]$ rpm -q --whatrequires httpd
Alternative way to check dependencies
3
Remove Orphaned Packages
Clean up packages that were dependencies but are no longer needed.
[admin@server ~]$ dnf autoremove
Show orphaned packages that can be removed
[admin@server ~]$ sudo dnf autoremove -y
Automatically remove all orphaned packages
4
Verify Removal
Confirm that packages were removed successfully.
[admin@server ~]$ rpm -q telnet
Check if telnet package is still installed
[admin@server ~]$ which telnet
Check if telnet command is still available
üí° What You Just Did
  • Used "dnf remove" to uninstall unwanted packages
  • Checked dependencies before removing critical packages
  • Used "dnf autoremove" to clean up orphaned dependencies
  • Verified removal with "rpm -q" and "which" commands
üì¶ Manage Package Groups
Install collections of related software as groups. This is useful for setting up development environments or security tool suites.
1
List Available Groups
See what package groups are available to install.
[admin@server ~]$ dnf group list
List all available package groups
[admin@server ~]$ dnf group list | grep -i development
Find development-related package groups
2
Get Group Information
See what packages are included in a group before installing.
[admin@server ~]$ dnf group info "Development Tools"
Show packages in Development Tools group
[admin@server ~]$ dnf group info "Security Tools"
Show packages in Security Tools group (if available)
3
Install Package Groups
Install entire groups of related software packages.
[admin@server ~]$ sudo dnf group install "Development Tools"
Install all development tools and compilers
[admin@server ~]$ sudo dnf group install -y "System Tools"
Install system administration tools automatically
4
Manage Installed Groups
Remove groups or check which groups are installed.
[admin@server ~]$ dnf group list --installed
List currently installed package groups
[admin@server ~]$ sudo dnf group remove "Development Tools"
Remove an entire package group
üí° What You Just Did
  • Used "dnf group list" to find available software collections
  • Used "dnf group info" to see what's included in a group
  • Installed complete tool suites with "dnf group install"
  • Managed installed groups and removed unwanted collections
üìú Package Installation History
Track what software has been installed, updated, or removed on your system. This is important for security auditing and troubleshooting.
1
View Transaction History
See a history of all package management operations.
[admin@server ~]$ dnf history
Show summary of recent package transactions
[admin@server ~]$ dnf history | head -10
Show last 10 package operations
2
Get Detailed Transaction Info
See detailed information about specific transactions.
[admin@server ~]$ dnf history info 5
Show details of transaction #5
[admin@server ~]$ dnf history info last
Show details of the most recent transaction
3
Check Installation Logs
View low-level package installation logs for troubleshooting.
[admin@server ~]$ tail /var/log/dnf.rpm.log
View recent package installation log entries
[admin@server ~]$ grep "$(date +%Y-%m-%d)" /var/log/dnf.rpm.log
Show today's package installation activity
4
Undo Transactions (Advanced)
Roll back package installations if something goes wrong.
[admin@server ~]$ dnf history undo last
Undo the most recent transaction (BE CAREFUL!)
[admin@server ~]$ dnf history undo 5
Undo transaction #5 (USE WITH EXTREME CAUTION!)
üí° What You Just Did
  • Used "dnf history" to track all package management operations
  • Used "dnf history info" to get detailed transaction information
  • Checked log files to see low-level installation details
  • Learned about "dnf history undo" for emergency rollbacks
